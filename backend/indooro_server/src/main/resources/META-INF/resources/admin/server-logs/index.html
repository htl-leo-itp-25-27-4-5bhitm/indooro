Letzte Fehler

Diese Seite zeigt protokollierte 4xx- und 5xx-Fehler aus der Backend-API. Fuer rohe Container-Logs verwendest du weiter kubectl logs in der LeoCloud.

CLI

Rohe LeoCloud Logs

Mit diesen Befehlen bekommst du die kompletten Backend-Logs direkt aus dem Cluster:

kubectl logs deployment/indooro-backend -n student-it220209 --tail=200
kubectl logs deployment/indooro-backend -n student-it220209 -f
kubectl get pods -n student-it220209
kubectl logs pod/<POD_NAME> -n student-it220209 --tail=200
