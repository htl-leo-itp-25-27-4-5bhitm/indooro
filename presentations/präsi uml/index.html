Was ist UML?

  • Unified Modeling Language
  • Standard zur Beschreibung von Software-Systemen
  • Visuelle Kommunikation zwischen Entwicklern
  • Unabhängig von Programmiersprache

Wozu verwendet man UML?

  • Systeme planen bevor Code entsteht
  • Komplexität reduzieren
  • Team-Kommunikation
  • Dokumentation
  • Architektur verstehen

Grenzen von UML

  • Kein Ersatz für Code
  • Runtime-Verhalten nur begrenzt sichtbar
  • Kann schnell zu detailliert werden
  • Pflegeaufwand

Typische UML Fehler

  • Klassen ≠ Objekte
  • Zu viele Beziehungen
  • Falsche Vererbung
  • Diagramm ohne Use-Case Bezug

Use Case 1: Produkt suchen → Klassen:
  • Product: Speichert Name und ID der Ware.
  • Shelf: Kennt die physische Position (x/y) im Markt.
  • SearchService: Logik, die die Datenbank durchsucht.
  • AppUI: Das Interface, wo der Kunde tippt.

Use Case 2: Route berechnen → Klassen:
  • Beacon: Liefert Bluetooth-Signal zur Ortung.
  • UserPosition: Speichert, wo der Kunde steht.
  • RouteCalculator: Algorithmus für den Weg.
  • StoreMap: Liefert Infos, wo Wände/Regale sind.

Projekt-Artefakte (Belege)

Vom UML zum echten System (Draft)

1. Java Klasse (aus dem Klassendiagramm) 
public class Product {
    public UUID id;
    public String name;
    public Shelf shelf; // Hier ist die Assoziation aus dem UML!
}
2. Datenbank-Schema (Entwurf) 
CREATE TABLE Product (
    id VARCHAR(50) PRIMARY KEY,
    name VARCHAR(100),
    shelf_id INT REFERENCES Shelf(id)
);

Diagramm Review (Fehlerkultur)

Typische Anfängerfehler am Beispiel Indooro

Fehler Vorher ❌ Nachher ✅ Erklärung
1. Vererbung statt Assoziation Beacon erbt von Position Beacon hat eine Position Ein Beacon *ist* keine Position. Er ist ein Gerät, das physisch dort montiert ist.
2. Objekte im Klassendiagramm Klasse heißt "Milch" Klasse heißt "Product" Klassen sind abstrakte Baupläne. "Milch" ist nur ein konkretes Objekt zur Laufzeit.
3. Falsche Abhängigkeit Product kennt Route Route kennt Product Die Milch im Regal weiß nicht, wer zu ihr navigiert.

KI – Pro & Contra

Pro
  • Schnelle Diagrammerstellung
  • Syntax Hilfe

Contra
  • Logische Fehler
  • Blindes Übernehmen

Zukunft von UML & KI

  • Text → UML automatisch
  • Skizze → Code
  • KI als Assistenz
  • Architektur bleibt menschlich
